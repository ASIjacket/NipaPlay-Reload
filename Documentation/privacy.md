# 隐私与数据

NipaPlay 尊重你的隐私。以下说明应用在不同场景下的数据处理方式。

## 本地数据

- 播放记录、设置、观看历史默认保存在本机（SQLite + SharedPreferences）。
- 不会在未经你操作的情况下上传到任何服务器。

## 网络请求

NipaPlay 在以下功能中会发起网络请求：

- **弹弹play**：匹配弹幕时会向弹弹play API 发送视频文件的哈希和文件名，用于获取匹配的弹幕数据。登录后会同步观看进度。
- **Bangumi**：配置访问令牌后，会与 bgm.tv API 通信以同步收藏、进度和评分。
- **Emby / Jellyfin**：连接媒体服务器时，身份认证和媒体元数据在你自己的服务器端处理。
- **AI 防剧透**：如果你启用了此功能，弹幕文本会被发送到你配置的 AI API 端点（OpenAI 兼容接口或 Google Gemini）进行分析。请确认你信任所配置的 API 服务商。
- **JS 插件**：第三方插件可能会发起网络请求，请注意审查插件来源。

## 日志

- 日志用于排障，可能包含文件路径、服务器地址等信息。
- 分享日志前请检查是否包含敏感信息。
